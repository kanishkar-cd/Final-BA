# Export Module

This module provides export functionality for user stories generated by the workflow pipeline.

## Architecture

The export module follows a clean separation of concerns:

1. **`formatter.py`** — Transformation layer
   - Accepts validated workflow output (UserStory objects, epics, features, traceability)
   - Converts to common internal model (`StoryExportData`)
   - No export logic — pure data transformation

2. **Exporters** — Format-specific export implementations
   - `word_export.py` — Word (.docx) export
   - `pdf_export.py` — PDF export
   - `jira_export.py` — Jira API integration
   - `confluence_export.py` — Confluence API integration

3. **`export_service.py`** — Main entry point
   - Dispatches to appropriate exporter
   - Handles configuration management

4. **`models.py`** — Data models
   - `StoryExportData` — Common story representation
   - `ExportRequest` / `ExportResponse` — API models
   - Configuration models for Jira/Confluence

5. **`utils.py`** — Shared utilities
   - Filename sanitization
   - Directory management
   - Text truncation

## Usage

### Step 1: Transform Workflow Output

```python
from export.formatter import WorkflowOutputFormatter

# After workflow completes, you have:
# - user_stories: list[UserStory]
# - epics: list[PlanningArtifact]
# - features: list[PlanningArtifact]
# - traceability: dict[str, Any]
# - export_metadata: dict[str, Any]

export_stories = WorkflowOutputFormatter.format_workflow_output(
    user_stories=workflow_response.stories,
    epics=workflow_request.epics,
    features=workflow_request.features,
    traceability=workflow_request.traceability,
    export_metadata={
        "project": "My Project",
        "assignee": "john.doe@example.com",
    }
)
```

### Step 2: Export to Desired Format

#### Word Export

```python
from export.export_service import ExportService
from export.models import ExportRequest, ExportFormat

service = ExportService()

request = ExportRequest(
    format=ExportFormat.WORD,
    stories=export_stories,
    project_name="My Project",
    include_metadata=True,
)

response = service.export(request)
print(f"Exported to: {response.file_path}")
```

#### PDF Export

```python
request = ExportRequest(
    format=ExportFormat.PDF,
    stories=export_stories,
    project_name="My Project",
    include_metadata=True,
)

response = service.export(request)
print(f"Exported to: {response.file_path}")
```

#### Jira Export

```python
from export.models import JiraExportConfig

jira_config = JiraExportConfig(
    project_key="PROJ",
    issue_type="Story",
    base_url="https://your-domain.atlassian.net",
    email="your-email@example.com",
    api_token="your-jira-api-token",
    assign_to_me=True,
)

request = ExportRequest(
    format=ExportFormat.JIRA,
    stories=export_stories,
    project_name="My Project",
)

response = service.export(request, jira_config=jira_config)
print(f"Created {response.story_count} Jira issues")
```

#### Confluence Export

```python
from export.models import ConfluenceExportConfig

confluence_config = ConfluenceExportConfig(
    space_key="PROJ",
    parent_page_id="123456",  # Optional
    base_url="https://your-domain.atlassian.net/wiki",
    email="your-email@example.com",
    api_token="your-confluence-api-token",
    include_toc=True,
    page_title_prefix="[User Story] ",
)

request = ExportRequest(
    format=ExportFormat.CONFLUENCE,
    stories=export_stories,
    project_name="My Project",
)

response = service.export(request, confluence_config=confluence_config)
print(f"Created {response.story_count} Confluence pages")
```

## Output Directories

- **`outputs/word/`** — Generated Word documents
- **`outputs/pdf/`** — Generated PDF documents

These directories are created automatically and include `.gitkeep` files to ensure they're tracked by Git.

## Extending the Module

### Adding a New Export Format

1. Create a new exporter class (e.g., `excel_export.py`)
2. Implement the `export(request: ExportRequest) -> ExportResponse` method
3. Add the format to `ExportFormat` enum in `models.py`
4. Register the exporter in `ExportService` in `export_service.py`

### Custom Formatting

Use `StoryFormatter` helper methods for consistent formatting:

```python
from export.formatter import StoryFormatter

# Format for different representations
plain_text = StoryFormatter.format_plain_text(story)
html = StoryFormatter.format_html(story)
markdown = StoryFormatter.format_markdown(story)
metadata = StoryFormatter.format_metadata_section(story)
```

## Integration with Backend

The export module is **independent** and does **not modify existing backend code**. 

To integrate:

1. Import the formatter in your workflow router
2. After workflow completion, transform the output
3. Call the export service with the desired format

Example integration in `workflow_router.py`:

```python
from export.formatter import WorkflowOutputFormatter
from export.export_service import ExportService
from export.models import ExportRequest, ExportFormat

@router.post("/workflow/export")
async def export_workflow(workflow_id: str, format: ExportFormat):
    # Fetch completed workflow
    workflow = await get_workflow(workflow_id)
    
    # Transform to export format
    export_stories = WorkflowOutputFormatter.format_workflow_output(
        user_stories=workflow.stories,
        epics=workflow.request.epics,
        features=workflow.request.features,
        traceability=workflow.request.traceability,
    )
    
    # Export
    service = ExportService()
    request = ExportRequest(
        format=format,
        stories=export_stories,
        project_name=workflow.project_name,
    )
    
    response = service.export(request)
    return response
```
